# 终端环境设置项目开发大纲

## 1. 项目概述

- **项目名称**: terminal-setup
- **项目类型**: 终端环境自动化配置工具
- **主要功能**: 一键配置完整的终端环境，支持多平台
- **目标用户**: 开发人员、系统管理员、终端用户
- **项目价值**: 减少手动配置时间，提供一致的终端体验

## 2. 支持平台

| 平台 | 状态 | 包管理器 |
|------|------|----------|
| 🍎 **macOS** | ✅ 主要支持 - 经过充分测试 | Homebrew |
| 🐧 **Debian / Ubuntu** | 🧪 实验性支持 - 可用但未广泛测试 | apt + 捆绑二进制文件 |
| 🪟 **Windows (WSL)** | 🧪 实验性支持 - 可用但未广泛测试 | apt (WSL内部) |
| 🪟 **Windows (原生)** | ⛔ 不支持 | 使用WSL替代 |

## 4. 安装流程

### 4.1 快速开始

#### macOS
```bash
git clone https://github.com/lewislulu/terminal-setup.git
cd terminal-setup && ./setup.sh
```

#### Debian / Ubuntu
```bash
git clone https://github.com/lewislulu/terminal-setup.git
cd terminal-setup && ./setup.sh
```

#### Windows (WSL)
1. 首先安装WSL:
```powershell
# 在PowerShell (管理员)中
wsl --install
```

2. 然后在WSL内部:
```bash
git clone https://github.com/lewislulu/terminal-setup.git
cd terminal-setup && ./setup.sh
```

### 4.2 安装选项

```bash
./setup.sh --fish       # Fish shell
./setup.sh --zsh        # Zsh + fish-like插件
./setup.sh --dry-run    # 预览将要执行的操作 (无更改)
```

### 4.3 一键安装 (自动克隆)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/lewislulu/terminal-setup/main/setup.sh)
```

## 5. 功能特性

### 5.1 Shell选择

| | 🐟 Fish | 🐚 Zsh |
|---|---------|--------|
| **POSIX** | ❌ 自有语法 | ✅ 兼容 |
| **自动建议** | ✅ 内置 | ✅ 通过插件 |
| **语法高亮** | ✅ 内置 | ✅ 通过插件 |
| **Node管理器** | fnm (共享) | fnm (共享) |
| **配置文件** | `~/.config/fish/config.fish` | `~/.zshrc` |
| **最适合** | 干净的默认设置，无麻烦 | 脚本编写，POSIX兼容 |

### 5.2 别名/缩写

| 快捷方式 | 展开为 |
|---------|--------|
| `ls` | `eza --icons --group-directories-first` |
| `ll` | `eza -la --icons --group-directories-first` |
| `lt` | `eza --tree --icons --level=2` |
| `cat` | `bat` |
| `find` | `fd` |
| `grep` | `rg` |
| `top` | `btop` |
| `lg` | `lazygit` |

### 5.3 fzf键绑定

| 按键 | 功能 |
|------|------|
| `Ctrl+R` | 模糊搜索命令历史 |
| `Ctrl+T` | 模糊查找文件 (使用 `fd` 作为后端) |
| `Alt+C` | 模糊cd到目录 |

### 5.4 fnm — Node版本管理器

```bash
fnm install 22            # 安装Node 22
fnm install --lts         # 安装最新LTS版本
fnm default 22            # 设置默认版本
fnm use 22                # 在当前shell中切换
 echo "22" > .node-version # 进入此目录时自动切换
```

### 5.5 SSH密钥切换器

两个shell配置都包含 `set-ssh-key` 函数，用于快速SSH密钥切换：

```bash
set-ssh-key my-key-name     # 清除代理，加载 ~/.ssh/my-key-name
set-ssh-key                  # 出错时显示可用密钥
```

## 6. 平台特定说明

### 6.1 macOS
- 完全支持，所有内容通过Homebrew安装
- Ghostty作为原生macOS应用安装

### 6.2 Debian / Ubuntu
- CLI工具在可用时通过apt安装，其他通过GitHub releases安装 (delta, lazygit, eza)
- `bat` → `batcat`, `fd` → `fdfind` — 自动创建符号链接
- 字体安装到 `~/.local/share/fonts/`
- Ghostty不在apt中 — 通过[snap, 从源码构建](https://ghostty.org/docs/install)手动安装，或使用其他终端
- Zsh插件通过apt或git clone安装

### 6.3 Windows (WSL)
- 所有操作在WSL内部运行 (Ubuntu/Debian层)
- 终端模拟器在Windows端运行 — 使用[Windows Terminal](https://aka.ms/terminal)或[Ghostty for Windows](https://ghostty.org)
- 脚本自动检测WSL并适应
- 如果在原生Windows (MINGW/Git Bash)中运行，脚本会提示安装WSL

## 7. 设计决策

### 7.1 为什么选择Ghostty?
- GPU加速，原生macOS应用，启动快速，配置格式简洁
- 现代、简约，不尝试做所有事情
- 仍处于早期阶段但发展迅速

### 7.2 为什么选择Starship而不是Powerlevel10k?
- **跨shell**: 相同的提示符配置可在Fish和Zsh中工作。p10k仅支持Zsh。
- **TOML配置**: 声明式且可读，而不是p10k的向导生成的混乱配置。
- **Rust二进制**: 快速，无shell框架依赖。
- **Catppuccin主题**: 与栈的其余部分一致。

### 7.3 为什么同时支持Fish和Zsh? 为什么不只选一个?
- **Fish**: 最佳开箱即用体验。自动建议、语法高亮、补全 — 全部内置，零配置。但它不兼容POSIX，因此 `bash` 脚本不能直接工作，一些工具假设POSIX shell语法。
- **Zsh**: 兼容POSIX，因此所有bash脚本和单行命令都能工作。通过插件 (autosuggestions + syntax-highlighting)，您可以获得Fish UX的90%。权衡：更多移动部件。

### 7.4 为什么使用Homebrew安装Zsh插件? 为什么不使用zinit/antidote/sheldon?
- 我们只安装3个Zsh插件：`zsh-autosuggestions`, `zsh-syntax-highlighting`, `zsh-completions`。
- 对于3个插件，插件管理器过度kill：
  - **zinit**: 原作者已放弃。社区分叉 (zdharma-continuum) 存在但增加了不必要的复杂性。
  - **antidote/sheldon**: 好工具，但仍然是额外的层。更多可能损坏的东西。
  - **Oh My Zsh**: 框架，不是插件管理器。启动时加载100+文件。慢。
  - **Homebrew**: 已经安装，`brew install` + `.zshrc` 中的一行 `source`。通过 `brew upgrade` 更新。零额外依赖。完成。

### 7.5 为什么选择fnm而不是nvm?
- **速度**: nvm增加200-400ms到shell启动。fnm增加~1ms。当您频繁打开终端时，这很重要。
- **统一**: 一个工具，一个Node安装位置 (`~/.local/share/fnm/`)，在Fish和Zsh中工作相同。使用nvm，您需要nvm-sh for Zsh和nvm.fish for Fish — 两个不同的工具，具有不兼容的存储路径。
- **自动切换**: `fnm env --use-on-cd` 读取 `.node-version` 或 `.nvmrc` 文件，并在您 `cd` 到项目时自动切换。不需要额外的钩子。

### 7.6 为什么选择MesloLGS NF?
- 专为终端使用设计 (等宽，小尺寸清晰)
- 包含所有Nerd Font符号 (图标，powerline，devicons)
- 与Powerlevel10k使用相同的字体 — 在终端中经过实战测试
- 提供Regular/Bold/Italic/Bold Italic版本

### 7.7 为什么选择git-delta for diffs?
- Git的默认diff输出功能齐全但丑陋。Delta添加：
  - diff中的语法高亮
  - 行号
  - 并排视图
  - 适当的单词级diff高亮
  - 使用 `n`/`N` 在文件之间导航

### 7.8 为什么选择zoxide over plain cd?
- zoxide学习您最常用的目录。几天后：
  ```bash
  z proj      # 跳转到 ~/projects (或任何您最常去的包含"proj"的路径)
  z doc       # 跳转到 ~/Documents
  zi          # 使用fzf进行交互式模糊选择
  ```

### 7.9 为什么选择fzf?
- 您可以安装的最有影响力的CLI工具：
  - **Ctrl+R**: 模糊搜索命令历史 (替代糟糕的默认reverse-i-search)
  - **Ctrl+T**: 通过模糊名称匹配查找任何文件
  - **Alt+C**: 通过模糊匹配cd到任何目录
  - 自动与 `fd` 集成 (比 `find` 快，尊重 `.gitignore`)
